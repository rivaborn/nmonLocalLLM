## Handoff Points: Debugging Guide

### src/nmon/config.py -> src/nmon/main.py
- **Function:** load_from_env() and load_persistent_settings()
- **Type in transit:** AppConfig
- **What to check:** Ensure config values are correctly parsed from environment and file
- **Failure mode:** Application uses default values instead of configured values

### src/nmon/main.py -> src/nmon/gpu/monitor.py
- **Function:** GpuMonitor.__init__()
- **Type in transit:** AppConfig, RingBuffer[GpuSample]
- **What to check:** Verify GPU device discovery and handle acquisition
- **Failure mode:** GPU monitoring fails to start or discovers no devices

### src/nmon/main.py -> src/nmon/llm/monitor.py
- **Function:** LlmMonitor.__init__()
- **Type in transit:** AppConfig, RingBuffer[LlmSample]
- **What to check:** Verify HTTP client initialization and Ollama API connectivity
- **Failure mode:** LLM monitoring fails to start or poll API

### src/nmon/gpu/monitor.py -> src/nmon/storage/ring_buffer.py
- **Function:** RingBuffer.append()
- **Type in transit:** GpuSample
- **What to check:** Verify samples are correctly stored in buffer with proper timestamps
- **Failure mode:** GPU data not available for UI rendering

### src/nmon/llm/monitor.py -> src/nmon/storage/ring_buffer.py
- **Function:** RingBuffer.append()
- **Type in transit:** LlmSample
- **What to check:** Verify samples are correctly stored in buffer with proper timestamps
- **Failure mode:** LLM data not available for UI rendering

### src/nmon/storage/ring_buffer.py -> src/nmon/ui/dashboard.py
- **Function:** RingBuffer.since(), RingBuffer.latest()
- **Type in transit:** List[GpuSample], List[LlmSample]
- **What to check:** Verify time-window queries return correct samples
- **Failure mode:** UI displays stale or incorrect data

### src/nmon/ui/app.py -> src/nmon/ui/dashboard.py
- **Function:** NmonApp.__init__()
- **Type in transit:** RingBuffer[GpuSample], RingBuffer[LlmSample]
- **What to check:** Verify buffer references are correctly passed to dashboard
- **Failure mode:** Dashboard cannot access GPU/LLM data for rendering
